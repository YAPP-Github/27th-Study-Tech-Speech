# Design-Pattern : Proxy

## 학습 링크
https://velog.io/@victoryu/Design-Pattern-Proxy
 
## 요약
1. 프록시는 실제 객체에 대한 접근을 제어하고 대변인 역할을 수행하는 구조적 디자인 패턴이다.

2. 실제 객체와 동일한 인터페이스를 구현하므로 클라이언트는 코드 수정 없이 객체의 생성 시점이나 사용 권한을 제어할 수 있다.

3. 지연 초기화, 보안, 로깅, 캐싱 등 비즈니스 로직 외의 부가적인 작업을 수행하여 시스템 자원을 효율적으로 관리하는 수단이다.
