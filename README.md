## 🎯 스터디 개요

### 목적
- **체계적 학습 정리**: 학습한 기술 내용을 문서화하여 지식 체계화
- **설명 역량 강화**: 복잡한 기술 개념을 명확하고 간결하게 설명하는 능력 훈련
- **커뮤니케이션 능력 향상**: 기술적 소통 능력 및 논리적 사고력 개발

### 기본 정보
- **주기**: 주 1회, 1시간 30분
- **스터디 시간 정하기**: 매주 **when2meet**에서 시간 조율
- **진행 방식**: Discord 비대면 온라인 또는 오프라인
- **기록 방식**: GitHub 레포지토리를 통한 학습 자료 및 발표 음성 텍스트 공유

## 📌 운영 원칙

### 1. 사전 준비
- 발표 **하루 전날 23:59**까지 학습 자료 커밋 완료
- 발표 자료는 반드시 **마크다운 형식**으로 작성
- 자료 내용은 마크다운 형식으로 작성하며, 외부 사이트 게시글 링크를 첨부하거나 핵심 내용을 직접 작성
- 발표 준비: 10분이란 제한된 시간동안 발표를 진행하기 때문에 시간에 알맞는 발표 길이 준비
- 예상 질의 답변 사전 준비: 나올 예상 질문들을 미리 조사하여 답변 대응 준비

### 2. 스터디 진행 방식
- 사전 탐색 10분 : 발표자별 **최소 2~3개 질문** 준비 필수
- 각 발표자별 **최대 15분** 할당 (발표 **최대 10분** + 질의응답 5분)
- 마무리 5분

### 3. 스터디 후 할 일
- 발표 및 Q&A 음성을 텍스트로 변환한 후 별도의 마크다운 파일 및 커밋 추가. 음성 텍스트를 통해 스피치 개선점 파악.
- 발표 내용과 질의응답에서 미흡했던 부분을 정리하여 학습 정리 파일 생성 후 추가 커밋.

## 🔄 세부 세션 진행 절차

### 학습 자료 커밋 마감
- **마감 시간**: 발표 하루 전날 23:59

### 제출 자료
1. **학습 정리 자료** (필수)
   - 파일명: `학습주제.md`
   - 형식: 마크다운
   - 내용: 학습한 기술 개념, 코드 예제, 참고 자료 등

2. **발표 및 Q&A 음성/텍스트** (발표 후 추가)
   - 파일명: `발표_학습주제.md`
   - 형식: STT 변환 텍스트의 마크다운
   - 내용: 실제 발표 내용 기록

3. **발표 및 질의응답 보완 자료** (발표 후 스터디 완료 뒤에 추가)
   - 파일명: `qna_학습주제.md`
   - 형식: 마크다운
   - 내용: 발표 내용과 질의응답에서 미흡했던 부분 보완

### 커밋 위치
```
weekN/참여자이름/학습주제.md
weekN/참여자이름/발표_학습주제.md
weekN/참여자이름/qna_학습주제.md
```
