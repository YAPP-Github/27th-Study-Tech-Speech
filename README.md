## 🎯 스터디 개요

### 목적
- **체계적 학습 정리**: 학습한 기술 내용을 문서화하여 지식 체계화
- **설명 역량 강화**: 복잡한 기술 개념을 명확하고 간결하게 설명하는 능력 훈련
- **커뮤니케이션 능력 향상**: 기술적 소통 능력 및 논리적 사고력 개발

### 기본 정보
- **주기**: 주 1회, 1시간
- **진행 방식**: Discord 비대면 온라인 또는 오프라인
- **기록 방식**: GitHub 레포지토리를 통한 학습 자료 및 발표 음성 텍스트 공유

## 📌 운영 원칙

### 1. 사전 준비
- 발표 **하루 전날 23:59**까지 학습 자료 커밋 완료
- 발표 자료는 반드시 **마크다운 형식**으로 작성
- 자료 내용은 마크다운에 직접 작성하거나 외부 사이트 게시글 링크를 첨부
- 발표 음성 STT 변환 텍스트를 발표 후 별도의 마크다운 파일에 추가

### 2. 스터디 진행 방식
- 사전 탐색 10분
- 각 발표자별 **최대 15분** 할당 (발표 3~10분 + 질의응답 5분)
- 마무리 5분

### 3. 적극적 참여
- 상대방 발표 자료를 사전에 탐색
- 발표자별 **최소 2~3개 질문** 준비 필수

### 4. 자기분석 및 성장
- 매 발표 후 자신의 발표 스타일, 답변 내용 등을 별도로 분석
- 발표 음성 텍스트를 통해 개선점 파악

## 🔄 세부 세션 진행 절차

### 📅 사전 준비 (발표 하루 전)

#### 커밋 마감
- **마감 시간**: 발표 하루 전날 23:59

#### 제출 자료
1. **학습 정리 자료** (필수)
   - 파일명: `학습주제.md`
   - 형식: 마크다운
   - 내용: 학습한 기술 개념, 코드 예제, 참고 자료 등

2. **발표 음성/텍스트** (발표 후 추가 가능)
   - 파일명: `발표_학습주제.md`
   - 형식: STT 변환 텍스트 또는 녹음 파일 링크
   - 내용: 실제 발표 내용 기록

#### 커밋 위치
```
weekN/참여자이름/학습주제.md
weekN/참여자이름/발표_학습주제.md
```
